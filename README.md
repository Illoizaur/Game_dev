### До 16 березня:
#### Мета: 
Ознайомитися з Unity, створити базову 2D-сцену з планетами.
#### Завдання: 
Ознайомтесь з описом сеансу гри (додано у док-файлі, де показано приклад для 100дронів), але ми будемо розробляти гру на 1000 дронів. \
Встановити Unity (остання LTS-версія) та створити новий 2D-проект під назвою "Cosmo Conquest. \
Створити сцену з фоном космосу: Завантажити безкоштовний асет космічного фону з Unity Asset Store (наприклад, "2D Space Background") або згенерувати через AI (запит: "2D cosmic background with stars and nebulae"). \
Додати 5 планет на сцену: Назви планет: Kronus (вулканічна), Lyrion (океанічна), Mystara (лісова), Eclipsia (темна), Fiora (квіткова). \
Використати безкоштовні асети планет із Unity Asset Store (наприклад, "2D Planets Pack") або згенерувати через AI (запит: "2D cartoon planet with [вулкани/океани/ліси тощо], vibrant colors"). \
Розташувати планети в рядок або півколо на сцені. \
Додати текст (UI Text) із назвами планет під кожною. \
#### Рекомендації: 
Використовуйте яскраві, контрастні кольори для планет (червоний Kronus, синій Lyrion, зелена Mystara тощо). \
Додайте легку анімацію обертання планет через Unity Animator для привабливості.
#### Результат:

### До 4 квітня:
#### Мета: 
Створити сервер для реєстрації та зберігання результатів.
#### Завдання: 
Налаштувати локальний сервер на PHP 8 (наприклад, https://laragon.org/ для віндовс, або XAMPP -https://www.apachefriends.org/.). /
Забезпечити можливість його доступу назовні, в інтернет (або через постійну ІР-адресу, або через тунелювання, якщо у вас динамічне ІР-  Ngrok:(з https://ngrok.com/) або Localtunnel (npm install -g localtunnel)
##### Створити базу даних: 
- Таблиця players (id, username, status).
- Таблиця games (game_id, player_id, kronus, lyrion, mystara, eclipsia, fiora, score).
##### Написати PHP-скрипти: 
- register.php: Реєстрація гравця (POST-запит із username, повертає player_id).
- start_game.php: Перевірка кількості зареєстрованих гравців (≥2), початок гри через 3 хвилини.
- submit_move.php: Збереження розподілу дронів гравця (POST-запит із player_id та чисел).
- get_results.php: Повернення результатів гри (JSON із балами всіх гравців).
#### Рекомендації: 
Використовуйте простий JSON-формат для обміну даними. Додайте базовий захист від SQL-ін’єкцій (mysqli_real_escape_string).
#### Результат: 
Сервер із базою даних і API для гри.
